# Ревью кода

Используй Read, Grep, Glob и Bash (только для `git diff`). Не редактируй файлы.

Проверь код по следующим критериям:

1. **Корректность** — логические ошибки, off-by-one, необработанные исключения
2. **Читаемость** — имена переменных, структура, комментарии где нужно
3. **Идиоматичность** — использует ли код идиомы своего языка там, где уместно (например, в Python — list comprehensions, context managers, pathlib)
4. **Безопасность** — SQL injection, hardcoded credentials, небезопасные операции с файлами
5. **Тестируемость** — можно ли это протестировать; есть ли side effects

Если ревьюишь текущие изменения — сначала запусти `git diff`.
Вывод: список находок, отсортированный по серьёзности (критичные → мелкие).
