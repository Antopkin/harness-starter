# cite — оформление библиографических ссылок

Собирает корректную ссылку на работу из её метаданных: запись BibTeX и/или
оформление по стилю (APA 7, ГОСТ Р 7.0.100–2018). Проверяет, что все нужные поля
на месте. Работает с одной ссылкой и со списком.

Обычно идёт после `lit-search` (нашли работы) или `pdf-digest` (законспектировали
PDF) — у них уже собраны метаданные каждой работы.

Таблица типов записей BibTeX, обязательных полей и развёрнутые примеры во всех трёх
форматах — в `references/bibtex-fields.md`. Загляни туда, когда нужен точный набор
полей под конкретный тип источника.

## Железное правило: не выдумывать библиографию

Ссылка существует, чтобы читатель нашёл источник. Выдуманное поле ломает саму цель.

- **Не сочиняй идентификаторы.** DOI, ISBN, номера страниц, том/выпуск — только те,
  что реально есть в метаданных. Нет — оставь пусто и пометь `% TODO: проверить`.
- **Не угадывай авторов и год.** Порядок и написание авторов — как в источнике.
  Сомневаешься в годе — не ставь «примерно».
- **Не подгоняй под красоту.** Лучше честная запись с пометкой о пропуске, чем
  полная, но наполовину придуманная.
- **Проверяй, а не доверяй.** Если DOI можно подтвердить (открыть `doi.org/<DOI>`
  или страницу работы) — подтверди. Автосгенерированную запись всё равно сверь
  глазами с источником.

## Рабочий цикл

1. **Собери метаданные.** Возьми из конспекта (`pdf-digest`), из карточки источника
   (`lit-search`) или с первой страницы PDF. Нужен минимум: авторы, год, название,
   тип источника (статья в журнале / доклад / книга / глава / препринт / веб).
2. **Определи тип записи** (см. таблицу в `references/bibtex-fields.md`):
   `@article`, `@inproceedings`, `@book`, `@incollection`, `@misc` (препринт/веб),
   `@techreport`, `@phdthesis` и т. д.
3. **Заполни поля** под тип. Обязательные — все; опциональные — если есть данные.
4. **Проверь полноту** (см. чек-лист). Пропуски помечай, не заполняй выдумкой.
5. **Оформи в нужном виде:** BibTeX и/или готовая строка в стиле (APA/ГОСТ).

## Откуда брать метаданные

- **Из PDF / конспекта** — первая страница статьи, шапка, футер с DOI.
- **По DOI** — открой `https://doi.org/<DOI>`; страница издателя даёт полные поля.
  Многие сайты дают готовый экспорт «Cite → BibTeX».
- **arXiv** — на странице работы есть блок цитирования и arXiv ID; для препринта
  это `@misc`/`@article` с полем `eprint`.
- **Веб-поиск** — если чего-то не хватает, найди страницу работы и добери поле.
- **Если подключены MCP-инструменты** (`zotero`, `paper-search`, `jina search_bibtex`)
  — используй их для автоподбора BibTeX и метаданных. Результат всё равно сверь с
  источником: авто-запись бывает неполной или с ошибками в авторах/страницах.

## Чек-лист полноты

Перед выдачей проверь по типу источника:
- **@article:** author, title, journal, year, volume, (number), pages, doi.
- **@inproceedings:** author, title, booktitle (название сборника/конференции),
  year, (pages), (publisher/organization), (doi).
- **@book:** author/editor, title, publisher, year, (address), (isbn).
- **@incollection** (глава): author, title, booktitle, editor, publisher, year,
  pages.
- **@misc** (препринт/веб): author, title, year, howpublished/url,
  (eprint+archivePrefix для arXiv), **urldate** (дата обращения) для веб-ресурсов.

Общее: у каждой записи — уникальный `citekey` (напр. `smith2024method`); имена в
BibTeX — `Фамилия, Имя` через ` and `; кириллицу в BibTeX оборачивай в `{…}`, чтобы
не сломался регистр; DOI — без `https://doi.org/`, только сам код в поле `doi`.

## APA 7 — кратко

Автор-год, «висячий» отступ, авторы через запятую с `&` перед последним, только
инициалы имён. Название статьи — обычным регистром (sentence case), название
журнала и том — курсивом.

- **Статья:** Фамилия, И. О., & Фамилия, И. О. (Год). Название статьи. *Название
  журнала, том*(выпуск), страницы. https://doi.org/DOI
- **Книга:** Фамилия, И. О. (Год). *Название книги* (изд.). Издательство.
- **Глава:** Фамилия, И. О. (Год). Название главы. В И. О. Фамилия (Ред.),
  *Название книги* (сс. X–Y). Издательство.

## ГОСТ Р 7.0.100–2018 — кратко

Русский стандарт для списков литературы. Область ответственности через `/`,
разделитель областей ` . — `, страницы через `С.`.

- **Статья:** Фамилия, И. О. Название статьи / И. О. Фамилия // Название журнала. —
  Год. — Т. X, № Y. — С. XX–YY.
- **Книга:** Фамилия, И. О. Название книги / И. О. Фамилия. — Город :
  Издательство, Год. — NNN с.
- **Электронный ресурс:** Автор. Название [Электронный ресурс]. — URL: адрес (дата
  обращения: ДД.ММ.ГГГГ).

Полные, развёрнутые примеры каждого типа — в `references/bibtex-fields.md`.

## Частые ошибки

- Перепутан порядок авторов или «И. О.» вместо реального имени — сверь с источником.
- `et al.` в самой записи списка литературы — в списке перечисляй всех авторов (или
  по правилу стиля), `et al.` уместно во внутритекстовой ссылке.
- DOI записан как ссылка внутри поля `doi` BibTeX — оставляй только код.
- Кириллица в BibTeX без `{}` — ломается регистр в некоторых стилях.
- Разные citekey у одной работы или дубли ключей — проверь уникальность.
- Для веб-ресурса нет даты обращения — добавь `urldate` / «(дата обращения: …)».

## Отвечай по-русски

Пояснения — на русском. Сами записи — на языке источника; для русскоязычного списка
по ГОСТ русские источники оставляй кириллицей. Технические токены (DOI, поля
BibTeX) — как есть.
