# transcript-verbatim

Verbatim-корректор ASR. Правит **ТОЛЬКО** ошибки машинного распознавания речи
(Deepgram nova-3); речь говорящих остаётся **дословной** — филлеры, повторы,
фальстарты, разговорный регистр и порядок слов сохраняются. Для интервью и
исследовательских расшифровок, где речь = данные.

## Вход и делегирование

Распознавание формата входа, сборка нейтрального `speaker_glossary`, модель
делегирования (один субагент на файл, один проход, батч по файлам), запрет
маркера континуации и вопрос «куда сохранить» — в
`skills/shared/transcript-io.md`. **НЕ дублировать** эти разделы; main линкует на
них и делегирует.

Маппинг нейтральных ролей в метки этого скилла: роли → `{Интервьюер}` /
`{Респондент}` (фокус-группа → `{Респондент 1}`, `{Респондент 2}`…). Fallback при
неопределимых ролях — `[Роли спикеров не определены автоматически]`.

## Инфостиль НЕ применять

Выход намеренно дословный. Инфостиль, чистка филлеров и формализация разрушат
речевой регистр и обесценят данные. Любые инструменты редактуры и чистки русского
текста (например, скилл `ru-text`) здесь **НЕ применять и НЕ рекомендовать**.

## Промпт субагенту

main делегирует ОДИН файл ОДНОМУ субагенту. В промпт заинлайнить правила ниже;
полную таксономию ошибок передать **ссылкой** на
`references/correction-rules.md`; `speaker_glossary` передать инлайном.

Поскольку субагент видит ВЕСЬ файл за один проход, rule 8 (системные ошибки →
один раз в логе с «повторяется N раз, таймкоды [первый]…[последний]») и
определение ролей работают сразу, без cross-chunk merge.

**8 кардинальных правил (инлайн):**

1. Сохранять ВСЕ филлеры: ну, вот, как бы, типа, значит, эээ, ммм, короче, блин — это данные, не мусор.
2. Сохранять ВСЕ повторы, самокоррекции, фальстарты, оборванные фразы.
3. Сохранять структуру речи: не разбивать, не объединять, не менять порядок слов.
4. Сохранять речевой регистр: разговорное оставлять разговорным, не формализовать.
5. Исправлять ТОЛЬКО ошибки МАШИНЫ: неверные слова, искажённые имена/названия, обрезанные слова, гендерные ошибки ASR.
6. Неуверенное помечать `[?предположение?]`, неразборчивое — `[неразборчиво]`.
7. Обработать ВЕСЬ файл за ОДИН проход. Маркер `[ПРОДОЛЖЕНИЕ СЛЕДУЕТ]` ЗАПРЕЩЁН (см. transcript-io.md §5).
8. Системная ошибка (одно слово неверно по всему тексту) → исправить ВСЕ вхождения, в логе записать ОДИН раз «(повторяется N раз)».

**Запрещено (forbidden_actions, кратко):** удалять филлеры, повторы или любые
фрагменты; реструктурировать или объединять предложения; полировать для
читаемости; формализовать разговорную речь; добавлять слова или логические связки;
менять пунктуацию (если она не явная ошибка ASR); исправлять грамматику
говорящего.

**Таксономия ошибок и все примеры** (proper_names, institutions, named_objects,
toponyms, abbreviations, word_errors, semantic_substitution — ОПАСНЫЙ ПАТТЕРН,
fused_words, truncated_words, gender_errors) → `references/correction-rules.md`.

**Форматы вывода** (шапка `=== ИНТЕРВЬЮ ===`, реплики `{Роль}:` без таймкодов,
`=== ЛОГ ИСПРАВЛЕНИЙ ===` с системными и единичными секциями) →
`references/correction-rules.md`.

## Вывод

В конце спросить пользователя, КУДА сохранить (см. transcript-io.md §6):

- (а) рядом с источником → транскрипт в `*_dialog_corrected.txt`, лог в `*_corrections.txt`;
- (б) по указанному пользователем пути;
- (в) только в чат.
