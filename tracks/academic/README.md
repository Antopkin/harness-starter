# Академический оверлей — трек «Работа с источниками»

> **Для человека (прочти эти три строки и всё).**
> Это академический оверлей для трека «Работа с источниками» — он ложится **поверх** базовой обвязки.
> База — в корне этого же репозитория. Сначала поставь её (в корне скажи агенту: «прочитай INSTALL.md
> и настрой обвязку под мой инструмент»), а потом — там же, в корне — скажи:
> **«прочитай tracks/academic/README.md и добавь академический оверлей»**. Дальше он всё сделает сам.

---

## Дальше — инструкция для агента

Ты — кодинговый агент (Claude Code, OpenCode или Codex). Тебя попросили «добавить
академический слой». Оверлей лежит в `tracks/academic/` этого репозитория; база
(`harness-starter`) — в его корне, и корень — твоя рабочая папка: все команды ниже
даны из корня. Это **надстройка** над уже установленной базовой обвязкой: она не
заменяет базовые правила и навыки, а добавляет поверх один слой — **дисциплину работы
с источниками** — плюс навыки, MCP-серверы и рецепты под этот слой.

Работай маленькими шагами. Перед тем как что-то копировать или править — покажи пользователю
план в одну-две строки. Ничего за пределами рабочей папки не трогай без спроса. Секреты и
чужие ключи — не читаем и не коммитим (правило из базового `AGENTS.md` действует).

### Шаг 0. Осмотрись — что лежит в оверлее

```
tracks/academic/           ← оверлей внутри репозитория базовой обвязки
├── README.md              ← этот файл (инструкция по установке оверлея)
├── AGENTS.academic.md     ← аддендум правил: честность источника, метод, воспроизводимость
├── skills/                ← академ-навыки, каталог на каждый: skills/<имя>/SKILL.md
│   └── shared/            ← общий файл ввода для семейства transcript-* (не отдельный навык)
├── mcp/
│   ├── README.md          ← как подключить два MCP-сервера (поиск статей + Zotero)
│   └── .mcp.json.example  ← шаблон конфига с плейсхолдерами вместо ключей
└── runbooks/              ← пошаговые рецепты трека
    ├── lit-review.md      ← обзор литературы от запроса до библиографии
    ├── eresources.md      ← платные э-ресурсы и библиотека вуза через браузер
    ├── rag.md             ← вопросы к своей папке PDF со ссылками на страницы
    └── zotero.md          ← поиск/заметки/аннотации в своей библиотеке Zotero
```

Если какого-то файла из списка нет — не выдумывай его, просто скажи пользователю,
чего не хватает, и продолжай с тем, что есть.

### Шаг 1. Убедись, что базовая обвязка уже стоит

Оверлей опирается на базу — она лежит в корне этого же репозитория. Проверь, что в корне
есть `AGENTS.md` (единый свод правил базы) — и, для Claude Code, `CLAUDE.md` со строкой
`@AGENTS.md`, а базовые навыки разложены по шагам корневого `INSTALL.md`.

- **Если база ещё не настроена** (навыки не разложены, `CLAUDE.md` не подключён) —
  остановись и попроси пользователя **сначала поставить базу** (в корне репозитория сказать
  агенту: «прочитай INSTALL.md и настрой обвязку под мой инструмент»). Без базы академический
  слой ставить некуда — накладывать нечего. Дождись базы и вернись к шагу 2.
- **Если база на месте** — продолжай.

### Шаг 2. Подключи правила трека (`AGENTS.academic.md`)

Аддендум `AGENTS.academic.md` читается **вместе** с базовым `AGENTS.md`, не вместо него.
Скопируй его из оверлея в корень — рядом с `AGENTS.md`:
`cp tracks/academic/AGENTS.academic.md .` Дальше — по инструменту:

- **OpenCode и Codex** читают файлы правил из корня **нативно**. Чтобы они гарантированно
  прочли и аддендум, добавь в конец базового `AGENTS.md` короткую строку-указатель, например:
  `> В академическом треке действует также AGENTS.academic.md — прочитай его вместе с этим файлом.`
- **Claude Code** сам файлы правил, кроме `CLAUDE.md`, не читает. Добавь в корневой `CLAUDE.md`
  строку `@AGENTS.academic.md` — прямо под строкой `@AGENTS.md`. Тогда аддендум подхватится
  вместе с базовыми правилами. Проверить, что реально загрузилось, можно командой `/memory`.

### Шаг 3. Подключи академ-навыки

Навыки трека лежат в `tracks/academic/skills/<имя>/SKILL.md`. У каждого во фронтматтере есть `name`
(совпадает с именем папки) и `description` (по нему инструмент понимает, когда навык
применить) — у всех наших навыков они уже проставлены. Рядом с навыками может лежать
`references/`, `templates/`, `agents/` — это их материалы, часть навыка; не выкидывай их.

Чтобы навыки заработали во **всех трёх** инструментах сразу, разложи их по двум путям —
`.claude/skills/` (Claude Code) и `.agents/skills/` (Codex). OpenCode отдельная копия не
нужна: он читает совместимые пути и подхватит любую из этих двух.

```
mkdir -p .claude/skills .agents/skills
cp -R tracks/academic/skills/. .claude/skills/
cp -R tracks/academic/skills/. .agents/skills/
```

Копирование **дополняет** уже стоящие базовые навыки, не затирает их. Как вызываются:

- **Claude Code** — по имени через `/` (`/deep-research`, `/paper-audit`, …) или автоподхватом
  по `description`. Нюанс: если папки `.claude/skills/` не было на момент старта сессии,
  Claude Code заметит новые навыки только после перезапуска.
- **OpenCode** — показывает агенту список навыков с их `description` через встроенный инструмент
  `skill` и сам вызывает подходящий по задаче (не через `/`).
- **Codex** — сам подбирает навык по совпадению задачи с `description`; список и ручной выбор —
  слэш-командой `/skills`.

**Если вдруг не подхватилось** (старая сборка инструмента) — деградируй мягко: открой нужный
`tracks/academic/skills/<имя>/SKILL.md` и иди по его шагам руками. Честность обеспечивают правила из
`AGENTS.academic.md`, а не тулза; навыки лишь ускоряют и стандартизируют.

### Шаг 4. Настрой два MCP-сервера

Оверлей подключает два внешних инструмента: **`paper-search`** (поиск и скачивание научных
статей по десяткам баз) и **`zotero`** (твоя личная библиотека). Полная пошаговая инструкция
и где брать ключи — в **`tracks/academic/mcp/README.md`**; готовый шаблон конфига —
**`tracks/academic/mcp/.mcp.json.example`**.

Коротко: поставь `uv` (для запуска серверов), скопируй шаблон в корень как `.mcp.json`
(`cp tracks/academic/mcp/.mcp.json.example .mcp.json`) и замени в нём `YOUR_..._KEY` / `YOUR_..._ID` на
**личные** ключи пользователя. Ключи оставляй **плейсхолдерами** — их подставит сам
пользователь; реальный ключ в `.mcp.json.example` не вписывай и `.mcp.json` в git не коммить.
Большинство баз `paper-search` работают вообще без ключа. Детали, локальный режим Zotero и
проверку подключения — смотри в `tracks/academic/mcp/README.md`.

### Шаг 5. Покажи, где рецепты (runbooks)

Скажи пользователю, что в `tracks/academic/runbooks/` лежат пошаговые рецепты трека —
каждый шаг можно копировать агенту почти как есть:

- `runbooks/lit-review.md` — обзор литературы: запрос → поиск → скрининг → PDF → конспект →
  синтез → ссылки → сохранение в Zotero.
- `runbooks/eresources.md` — доступ к платным ресурсам и библиотеке вуза через браузер,
  в правовой рамке.
- `runbooks/rag.md` — вопросы к своей папке PDF (`materials/`) с ответами со ссылками на страницы.
- `runbooks/zotero.md` — искать, сохранять заметки и аннотировать в своей библиотеке через агента.

### Шаг 6. Объясни человеку, что добавилось

Коротко и без жаргона скажи пользователю примерно следующее (своими словами):

- Правила теперь строже к источникам: **цитируй всё, ничего не выдумывай**, отделяй
  «что сказано» от «откуда это», фиксируй след поиска (правила — в `AGENTS.academic.md`).
- Появились большие навыки трека: **`deep-research`** (строгое исследование под вопрос —
  систематический обзор, мета-анализ, fact-check), **`academic-paper`** (написать саму статью),
  **`academic-paper-reviewer`** и **`paper-audit`** (прогнать черновик через «рецензентов» и
  проверить честность), **`transcript-verbatim`/`transcript-polish`** (расшифровки интервью,
  где речь — это данные).
- Агент теперь умеет **искать статьи по десяткам научных баз** и **работать с твоей библиотекой
  Zotero** — через два MCP-сервера (когда подставишь ключи).
- Работа с PDF и цитатами и обзор литературы разложены по рецептам в `tracks/academic/runbooks/`.
- Одной фразой: **«трек включён»** — агент держит дисциплину источников, умеет исследовать,
  писать и рецензировать, ищет по базам и помнит твою библиотеку.

## Навыки трека

| Навык | Для чего | Когда звать |
|---|---|---|
| `deep-research` | Строгое исследование под вопрос: систематический обзор, мета-анализ, fact-check, синтез многих источников | Большой исследовательский вопрос, нужен протокол и обзор литературы |
| `academic-paper` | Написание самой статьи (IMRaD, обзор, кейс, policy brief), цитаты, двуязычный abstract | Материал собран — пора писать текст |
| `academic-paper-reviewer` | Симуляция рецензирования: несколько независимых рецензентов + редакторское решение | Перед подачей — прогнать черновик через «рецензентов» |
| `paper-audit` | Аудит целостности: тезисы без источника, выдуманные цитаты, запрещённые формулировки | Проверка честности готового текста перед сдачей |
| `transcript-verbatim` | Verbatim-корректура ASR: править только ошибки распознавания, речь оставлять дословно | Расшифровка интервью/фокус-группы, где **речь = данные** |
| `transcript-polish` | Читаемый связный текст из устной речи с сохранением смысла и намерений | Транскрипт нужно сделать читаемым для анализа |
| `academic-pipeline` | Оркестрация полного конвейера: research → write → integrity check → review → revise → re-review → finalize | Нужно провести статью по всем стадиям одним прогоном, не собирая шаги вручную |
| `latex-fix` | Исправление ошибок компиляции LaTeX по анализу log-файла | Не собирается `.tex` — ошибки компиляции, недостающие пакеты, битая библиография |
| `latex-paper-en` | Ассистент по готовой англоязычной LaTeX-статье: компиляция, грамматика, библиография, перевод, разделы эксперимента | Статья на английском под IEEE/ACM/Springer/NeurIPS/ICML |
| `latex-proofread` | Двухфазный пруфридинг LaTeX-статьи: инфраструктура (preamble, макросы, cross-ref, цитаты, фигуры) + контент (грамматика, ясность, нарратив) | Финальная вычитка перед подачей на конференцию уровня ICRA/RSS/NeurIPS/T-RO/CVPR |

Хребет работы с литературой — навыки `lit-search`, `pdf-digest`, `cite` — уже стоит из
базовой обвязки; оверлей добавляет навыки крупнее. Каждый навык — это папка
`tracks/academic/skills/<имя>/` с файлом `SKILL.md`; открой любой `SKILL.md`, чтобы увидеть,
что он делает и по каким триггерам срабатывает. Семейство `transcript-*` опирается на общий
файл `skills/shared/transcript-io.md` (он копируется вместе с навыками).
