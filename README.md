# harness-starter — стартовая обвязка для кодингового агента

Обвязка (harness) — это то, что превращает кодингового агента из чат-бота в дисциплинированного
работника. Она состоит из трёх вещей: **правила** (как себя вести), **навыки** (готовые
процедуры под частые задачи) и **память** (факты, переживающие перезапуски сессий).
Этот репозиторий — готовый стартовый набор всех трёх, работающий в **Claude Code, OpenCode
и Codex** из одной раскладки файлов.

Собран для летней школы по AI-native работе, но пригоден любому, кто начинает работать
с кодинговыми агентами: клонируешь — и обвязка сразу под рукой.

> **Агенту.** Если тебя попросили «настроить обвязку» — открой `INSTALL.md` и следуй ему.

## Быстрый старт

```
git clone <URL этого репозитория>
cd harness-starter
```

1. Открой в этой папке своего агента — Claude Code, OpenCode или Codex.
2. Скажи ему: **«прочитай INSTALL.md и настрой обвязку под мой инструмент»**.
3. Агент сам определит инструмент, разложит правила и навыки куда нужно и объяснит,
   что получилось.
4. Проверь установку первым заданием из `hello.md` — это пара минут.

## Что внутри

| Что | Где | Зачем |
|---|---|---|
| Правила | `AGENTS.md` (+ обёртка `CLAUDE.md`) | Единый свод: думать до кода, минимальные правки, доказывать результат, не трогать секреты |
| Навыки | `skills/` — 13 штук | Готовые процедуры под частые задачи, см. список ниже |
| Память | `memory/MEMORY.md` | Индекс фактов, которые агент помнит между сессиями |
| Инсталлятор | `INSTALL.md` | Пошаговая инструкция для самого агента |
| Первое задание | `hello.md` | Проверка за минуту, что всё подключилось |
| Рецепты | `runbooks.md` | Частые операции: браузер к ресурсу, откат правки, свой навык |
| Треки | `tracks/` | Надстройки под направление; сейчас один — академический |

### Навыки базового набора

- **Код:** `review` (ревью), `test` (pytest-тесты), `tdd` (Red-Green-Refactor),
  `diagnose` (отладка трудных багов), `explain` (объяснение кода).
- **Текст:** `ru-text` (русская типографика и редактура), `writing-guru` (нарративная
  стратегия), `style-extract` (профиль авторского стиля).
- **Источники:** `lit-search` (поиск литературы), `pdf-digest` (конспект PDF),
  `cite` (оформление ссылок).
- **Мета:** `skill-creator` — создать и отладить собственный навык.
- **Процесс:** `handoff` — компактная передача работы следующей сессии.

Полная таблица с триггерами — в `INSTALL.md`, раздел «Навыки в стартере».

## Матрица инструментов

Правила и навыки по содержанию одни на всех — различается только раскладка файлов,
и её агент делает сам по `INSTALL.md`:

| Инструмент | Правила | Навыки (папка) | Вызов навыка |
|---|---|---|---|
| **Claude Code** | `CLAUDE.md` → строка `@AGENTS.md` | `.claude/skills/` | `/имя` или автоподхват по описанию |
| **OpenCode** | `AGENTS.md` нативно | читает и `.claude/skills/`, и `.agents/skills/` — своя копия не нужна | встроенный инструмент `skill` |
| **Codex** | `AGENTS.md` нативно | `.agents/skills/` | автоподхват + список `/skills` |

## Треки

**[`tracks/academic/`](tracks/academic/README.md)** — академический оверлей поверх базы,
трек «Работа с источниками»: аддендум правил честности источника (`AGENTS.academic.md`),
шесть крупных навыков (`deep-research`, `academic-paper`, `academic-paper-reviewer`,
`paper-audit`, `transcript-verbatim`, `transcript-polish`), два MCP-сервера (поиск статей
по десяткам научных баз + личная библиотека Zotero) и пошаговые рецепты в `runbooks/`.

Установка: сначала база (см. быстрый старт), затем скажи агенту в корне репозитория:
**«прочитай tracks/academic/README.md и добавь академический оверлей»**.
